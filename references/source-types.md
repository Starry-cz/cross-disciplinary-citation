# 来源类型与最小元数据

先确定“被引用的智力成果”而非只看载体。记录实际阅读的版本、位置与持久标识符。

| 来源类型 | 最小元数据 | 额外核对 |
| --- | --- | --- |
| 期刊文章 | 作者、题名、期刊、年、卷(期)、页码或文章号 | DOI、勘误/撤稿状态、online-first 与正式版关系 |
| 会议论文/摘要 | 作者、题名、会议或论文集、年、页码/文章号 | 是同行评议论文、海报、摘要还是演示文稿 |
| 专著 | 作者/编者、书名、版次、出版者、年 | 译者、卷册、电子版/印刷版、章节范围 |
| 书中章节 | 章节作者与题名、编者、书名、页码、出版者、年 | 章节不是整本书；不要混淆章节作者和编者 |
| 学位论文 | 作者、题名、学位类型、授予机构、年 | 数据库链接或机构库持久链接 |
| 技术/政府/机构报告 | 机构或作者、题名、报告号、机构、年 | 版本、发布日期、永久链接 |
| 预印本/工作论文 | 作者、题名、预印本服务器/系列、版本、年、标识符 | 是否已有正式发表版；不要双重计数 |
| 数据集 | 作者/创建者、题名、版本、存储库、年、DOI/持久标识 | 数据发布日期、许可、accession number |
| 软件/代码仓库 | 作者/维护者、软件名、版本或 release、发布者/仓库、年、DOI/URL | 优先 `CITATION.cff`、release/tag 或 commit |
| 标准/规范 | 机构、标准号、题名、版本/年份、发布者 | 勘误、采用版本与法域 |
| 专利 | 发明人、题名、专利号、授予机构/国家、日期 | 申请号与公开/授权状态 |
| 网页/在线文档 | 作者或机构、页面题名、网站、发布日期/更新日、URL | 内容是否易变；访问日期是否被目标规范要求 |
| 法规/判例 | 法域、案名或法名、法院/立法机关、年份、报告/中立引注、页码或段落号 | 只按适用法域指南格式化 |
| 视听/图像/社媒 | 创作者、题名/描述、平台或载体、日期、URL/永久标识 | 时间戳、版本、版权和是否允许复用 |
| 个人通信/访谈 | 通信者、形式、日期、地点/存档信息（若适用） | 多数体系只在正文/注释中引用，不进文后书目 |

## 通用字段原则

- DOI 用其规范形式（通常是 `https://doi.org/...`）；不要把 DOI、ISBN、PMID、arXiv 号、专利号和 URL 混写为同一字段。
- 作者、组织、编辑者、翻译者和软件维护者承担的角色不同，须据目标体系标明或排序。
- 网页的访问日期不是发布日期的替代物；仅在目标规范要求或内容易变且规则允许时加入。
- 不把数据库检索页、搜索引擎缓存或二手书目信息当作原始文献链接。
